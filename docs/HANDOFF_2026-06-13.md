## Strategy economics learned (NOT bugs)
- **Fees ≈ the edge** on 5-lot BTC round-trips — each filled leg pays Delta fees; cancelled IOC rungs
  are free. Winners ~+0.075 USD vs ~0.06–0.10 USD fees/round-trip. Needs fewer/bigger/wider-target trades.
- **Balanced-pair selection biases to the cheapest far-OTM pair** (normalized premium diff) → lands on
  exit thresholds. Mitigated by raising the crypto entry **theta floor (30 → 50)**. Open option A/B
  (deferred, user deciding): A = blend credit into the score; B = ATM-distance tie-break. Implement
  behind a crypto-only flag so NSE stays byte-identical.
- **Trailing SL semantics**: `Lock` = activation peak%, `Floor` = give-back distance below peak →
  exit at `peak − Floor`. MUST set **Floor ≤ Lock** or it exits below break-even. User's correct
  setup = Lock 20 / Floor 10–12.5 (locks ≥ +7.5–10%, trails up).
- **PnL guardrail SL must be NEGATIVE** (e.g. −100 pts) — it's the CUMULATIVE day stop
  (`session_realized + running ≤ SL`). A positive value insta-fires (0 ≤ +100).
- **Day-SL is now CUMULATIVE on BOTH bases** (fixed 2026-06-13): `total_day_pct =
  (session_realized_pnl_pts + running) / initial_net_credit × 100`. Previously the THETA path used
  `pos.premium_decay_pct()` = current position only, so a "Day" SL was really per-position (one trade
  doubling tripped it; a string of small losers never would). Now a "Day SL X%" caps the WHOLE day on
  theta too, matching the LTP basis and the user's intent. The **trailing SL still uses per-position
  `premium_decay_pct()`** (correct for a trail). Because the theta Day-% is now the cumulative cap, the
  user **disabled the PnL guardrail** (it was the redundant cumulative stop). NOTE: theta and ltp day-%
  are now nearly identical (premium_decay_pct numerator was already LTP P&L, not intrinsic-stripped).
- Crypto **session resets at 18:30 IST** (entry_start). `stop_for_day`/`trades_today` persist within a
  session across restarts; `scripts/fresh_start.sh BTC` wipes to test now.
